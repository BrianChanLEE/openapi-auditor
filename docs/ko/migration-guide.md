# Migration Guide: openapi-auditor v1.x

이 문서는 openapi-auditor v1.0.0에서 최신 고도화 버전(v1.2.0+)으로 전환하는 사용자를 위한 가이드입니다.

## 1. 주요 변경 사항 요약 (No-Emoji)
- **스키마 버전 도입**: `REPORT.json`에 `schemaVersion` 필드가 추가되었습니다.
- **성능 지표 세분화**: 평균 응답 시간 외에 P95, P99 지표가 기본 산출됩니다.
- **우선순위 엔진 개편**: 하드코딩된 로직이 `RuleEngine` 모듈로 분리되었으며 외부 규칙 주입이 가능합니다.
- **이모지 제거**: 모든 출력물 및 리포트에서 이모지가 텍스트 배지로 대체되었습니다.

## 2. REPORT.json 호환성
v1.1.0 이상의 스키마는 하위 호환성을 유지합니다. 기존 `summary` 필드 구조는 유지되면서 아래 필드들이 새롭게 추가되었습니다.
- `summary.performance`: P50~P99 상세 분포 데이터
- `coverage`: 엔드포인트 및 권한별 커버리지 지표

## 3. 신규 CLI 옵션 활용
- **--compare <path>**: 이전 리포트와 비교하여 회귀(regression)를 자동으로 감지합니다.
- **--trace-rules**: 각 엔드포인트에 어떤 진단 규칙이 적용되었는지 추적 로그를 출력합니다.

## 4. 하위 호환성 정책
- 기존에 사용하던 모든 CLI 인자명과 기본 동작은 그대로 유지됩니다.
- 스키마 버전이 없는 이전 JSON 리포트는 자동으로 v1.0.0 호환 모드로 해석됩니다.
