# 문제 해결 (Troubleshooting)

자주 발생하는 문제들과 해결 방법입니다.

## 1. OpenAPI 스펙 로드 실패
- **현상**: `ERROR: OpenAPI Load Failed`.
- **원인**: 파일 경로가 잘못되었거나 문서에 문법 오류가 있습니다.
- **해결**: 
  - `npx openapi-auditor validate <path>` 명령어로 스펙 유효성을 먼저 확인하세요.
  - URL을 사용하는 경우 공개적으로 접근 가능한지 확인하세요.

## 2. 모든 요청이 401/403 발생
- **현상**: 실패율 100% 및 `AUTH` 카테고리 발생.
- **원인**: 대상 서버가 진단 도구의 JWT 시크릿이나 토큰 형식을 인식하지 못합니다.
- **해결**: 
  - 진단 세션 동안 서버가 `test-secret-key`(기본값)를 사용하도록 설정하세요.
  - Proxy나 Gatekeeper에서 `Authorization` 헤더를 제거하고 있지는 않은지 확인하세요.

## 3. 서버가 500 오류를 반환
- **현상**: 다수의 `SERVER_ERR` (P0) 이슈 발생.
- **원인**: 생성된 페이로드가 예기치 않은 크래시를 유발하거나 데이터베이스가 초기화되지 않았습니다.
- **해결**: 
  - 리포트의 "Data" 섹션에서 전송된 실제 페이로드를 확인하세요.
  - 서버의 애플리케이션 로그와 스택 트래이스를 분석하세요.

## 4. 타임아웃 및 네트워크 오류
- **현상**: `NETWORK` 카테고리 실패 발생.
- **원인**: `baseUrl`이 잘못되었거나 서버가 너무 느리거나 오프라인 상태입니다.
- **해결**: 
  - 터미널에서 `curl <url>` 등을 이용해 `baseUrl`에 도달 가능한지 확인하세요.
  - `api-quality-config.json`에서 타임아웃 값을 늘려보세요.

## 5. 리포트 폴더가 생성되지 않음
- **현상**: `./reports/REPORT.md`를 찾을 수 없음.
- **원인**: 쓰기 권한 문제이거나 프로세스가 조기 종료되었습니다.
- **해결**: 진단 도구가 현재 디렉토리에 파일을 쓸 수 있는 권한이 있는지 확인하세요.
