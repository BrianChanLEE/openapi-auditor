# 리포트 형식 (`REPORT.md`)

자동으로 생성되는 `REPORT.md`는 이해관계자 및 개발자와 공유하여 즉각적인 실행 계획(Action Plan)으로 사용할 수 있도록 설계되었습니다.

## 리포트 구조

### 1. 종합 요약 (Overall Summary)
API 건강 상태에 대한 고수준의 뷰를 제공합니다.
- **성공률(Success Rate)**: 계약 스펙 및 성능 기준을 모두 충족한 요청의 백분율.
- **평균 지연 시간(Average Latency)**: API의 전반적인 응답 속도.
- **이슈 카운트**: P0, P1, P2, P3 등급별 이슈 발생 건수.

### 2. 주요 이슈 (Priority Issues, P0 ~ P1)
즉각적인 수정이 필요한 치명적인 버그들을 나열합니다.
- **P0 예시**: 서버 크래시(500), 보안 허점(READONLY 권한으로 POST 성공).
- **P1 예시**: 유효한 Admin 계정의 인증 실패(401/403).

### 3. 진단 카드 (Diagnosis Cards)
각 실패 건에 대한 상세 분석 카드를 생성합니다.
- **엔드포인트**: 메서드 및 경로.
- **역할(Role)**: 테스트된 유저 권한.
- **원인(Reason)**: 근본 원인 (예: "네트워크 타임아웃", "401 권한 없음").
- **수정 가이드(Fix Guidance)**: 이슈 해결을 위한 구체적인 액션 아이템.

### 4. 리스크 분석 테이블
우선순위 및 역할을 기준으로 정렬된 모든 테스트 케이스 요약 테이블입니다.

## 리포트 활용 방법
1. **CI/CD 통합**: 자동화된 배포 후 리포트를 검토하여 품질 게이트로 활용하세요.
2. **이슈 트래킹**: "수정 가이드" 텍스트를 Jira나 GitHub Issue의 설명으로 바로 활용하세요.
3. **협업**: 백엔드 팀과 MD 파일을 공유하여 상세 로그를 기반으로 빠른 디버깅을 진행하세요.
